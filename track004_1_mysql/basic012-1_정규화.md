■ 진행1. 정규화 1NF ~ 5NF
STEP1) 정규화?
1. 관계형 데이터 모델 - 데이터간 관계에 초점을 둔 모델
2. 이상 (Anomaly)   - 공간낭비
   - 삽입, 갱신, 삭제 이상
3. 정규화? 이상을 제거


STEP2) ERD 핵심구성요소
1. 개체(Entity) - 테이블 , 개념의 정보단위
2. 속성(Attribute) - 테이블의 열 , 가장 작은 논리적 단위
3. 관계(Relationship) - 외래키, 테이블 간에 관계를 외래키 등을 구현해서 사용

예) emp( FK:deptno ) - dept( PK:deptno )

STEP3.5)  BCNF
1. 관계형데이터베이스의 X → Y 성립할때 
   모든 결정자가 후보키이면 BCNF가 된다.
2. 식별자로 쓰이는 속성이 일반속성에 종속되지 않아야 한다.
3. 3NF를 통해서도 제거되지 않은 3.5 정규화

※ KEY?
: 튜플( 행, 레코드 ) 들을 구분할수 있는 기준필드
NO(pk)   NAME    AGE     jumin
1       장동건    54     720514
2       장동건    54     721214

1) 후보키 - 기본키로 사용할수 있는 속성 (no, jumin)
    유일성(유일하게 식별), 최소성(식별시 최소한의 속성)
2) 기본키 - 후보키에서 선택한 주키(main key) - no
3) 대체키 - 후보키엣어 선택 안된 키 - jumin
4) 외래키 - 테이블과 테이블을 연결해주는 속성 (deptno)
5) 슈퍼키 - 속성+속성 집한된 구성키
   std_id  + course_id
   유일성o( 다른 튜플 구분o), 최소성(x)
