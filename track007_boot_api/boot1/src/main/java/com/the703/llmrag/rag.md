#   RestClient와 Apache PDFBox를 활용한 파일 기반 RAG 생성형 AI 서비스 개발

---

##   Part 1. 생성형 AI 핵심 이론: RAG (검색 증강 생성)

### 1. RAG(Retrieval-Augmented Generation)의 개념

RAG는 "AI에게 외부 지식을 검색할 수 있는 참고서(오픈북)를 쥐여주는 기술"입니다.

일반적인 LLM(ChatGPT 등)은 사전 학습된 데이터 내부에서만 답변하므로, 사내 문서나 실시간 최신 정보에 답할 수 없으며 거짓말을 지어내는 환각 현상(Hallucination)이 발생합니다. RAG는 이러한 단점을 보완하기 위해 "사용자의 질문에 부합하는 문서 내용을 먼저 찾은 뒤(Retrieval), 그 내용을 프롬프트에 결합하여(Augmentation), AI가 답변을 생성(Generation)하도록 유도"하는 기법입니다.

### 2. RAG의 핵심 장점

* **환각(Hallucination) 방지:** 주어진 문서 안에서만 근거를 찾아 답변하므로 무분별한 뇌피셜(거짓 답변)을 막습니다.
* **최신 정보 및 사내 데이터 활용:** 모델을 매번 새로 재학습(Fine-tuning)시키지 않고도, 실시간 업로드된 텍스트나 PDF 내용을 답변에 즉시 반영할 수 있습니다.
* **비용 최적화 및 보안:** 거대한 인프라 구축이나 복잡한 파이썬 LLM 스택(LangChain 등) 없이, 자바 백엔드 생태계 내부에서 경량 아키텍처로 구현이 가능합니다.

### 3. RAG의 3단계 작동 매커니즘

```
[사용자 질문] ➔ [1. 검색 (Retrieval)] ➔ [2. 증강 (Augmented)] ➔ [3. 생성 (Generation)]
                     └─ PDF 텍스트 추출         └─ 질문 + 문서 결합         └─ OpenAI API 호출

```

### 4. 유의사항 
> 실습용 PDF 문서는 **1~2장 내외의 가벼운 이력서나 사내 규정집**을 권장 
파일이 지나치게 두꺼우면 OpenAI API 토큰 제한량(Context Window)을 초과하여 요금 폭탄이나 예외가 발생할 수 있음

##   Part 2. 면접 전략 및 포트폴리오 가이드

단순 쇼핑몰 CRUD 클론 코딩 포트폴리오들을 압도할 수 있는 무기가 됩니다.

###   왜 이 방식이 면접에서 더 높은 점수를 받는가?

1. **프레임워크 뒤에 숨은 HTTP 아키텍처 이해:** `Spring AI` 프레임워크를 그냥 가져다 쓰는 주니어들과 달리, 
**공식 API 스펙 문서를 읽고 원시 데이터 통신 구조와 DTO 프로토콜을 직접 설계한 기본기**를 입증합니다.

2. **비정형 데이터의 데이터 파싱 역량:** `Apache PDFBox`와 스프링의 멀티파트 스트림 처리를 연동하여, 
가공되지 않은 문서 데이터를 정제하고 비즈니스 컨텍스트(Context)로 조립하는 실무 역량을 보여줍니다.

3. **독립적이고 안정적인 경량 아키텍처:** 외부 의존성(Dependency)을 최소화하여 프레임워크 버전 변동 리스크를 줄이고 스프링 고유 기능만으로 유지보수 비용을 낮추는 **클린 코드 및 아키텍처 설계 관점**을 면접에서 강하게 소구할 수 있습니다.

### 📝  이력서용 기술 기술  

* **프로젝트 설명:**
> *"Spring 6의 `RestClient` 아키텍처를 독립적으로 설계하여 외부 OpenAI 생성형 모델 연동 레이어를 구축하였으며, `Apache PDFBox` 스트림 제어를 결합한 인메모리 방식의 경량 RAG(검색 증강 생성) 모듈을 구현했습니다."*

* **주요 해결 과제:**
> *"타사 AI 라이브러리 도입에 따른 의존성 비대화 및 버전 충돌 리스크를 전면 배제하기 위해 직접 Chat Completion 프로토콜 매핑용 응답 DTO 레코드를 설계하여 시스템 결합도를 최소화했습니다."*
